# 对话记录 — 2026-04-13

## 问题背景

陛下邮箱有 5000+ 封邮件，但扫描只找到 46 封。

---

## 问题一：为什么只扫出 46 封？

**原因：** 默认命令 `python main.py scan` 只扫描"促销邮件（category:promotions）"，
而且只看最近 30 天。很多垃圾邮件被 Gmail 归到其他分类，所以没被扫到。

**解决：**
```bash
python main.py scan --all          # 扫所有分类
python main.py scan --all --days 0  # 扫所有分类 + 全部历史邮件（不限时间）
```

---

## 问题二：大量 429 错误，很多邮件被跳过

**错误内容：** `Too many concurrent requests for user`

**原因：** 原来的代码用 Gmail Batch API，会把 50 封请求"打包"同时发给服务器，
服务器端并发执行，触发 Gmail 对单用户的并发连接上限，导致 429 报错并丢失数据。

**修复：** 改为逐封顺序请求，每封之间休眠 0.12 秒（约 8 req/s），遇到 429 自动重试退避。

**修改文件：** `scanner.py` — `_fetch_messages_batch` 函数完全重写

---

## 问题三：如何扫全部历史邮件（不限时间）？

**修复：** 增加 `--days 0` 作为"哨兵值"，传入 0 时不加 `after:` 时间过滤。

```bash
python main.py scan --all --days 0
```

`main.py` 的 `--days` 帮助文字也已更新：`0 = 不限时间扫全部`

---

## 问题四：扫描速度由什么决定？如何加速？

**决定因素：** 是**代码**，不是电脑配置。
0.12s 的等待是代码里人为加的，目的是避免触发 Gmail 速率限制。
5000 封邮件约需 10 分钟。

**加速方案（待实现）：**
3-5 个并发请求 + 429 自动重试，预计提速 3 倍（约 3-4 分钟）。
陛下说"等这次扫描完再做"，下次继续时可以直接开始实现。
